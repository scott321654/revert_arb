# TW50 Rebalance

台灣 50 指數（0050）成分股季度調整 — 尾盤均值回歸策略。

## 策略原理

台灣 50 指數每年 3/6/9/12 月進行季度審核，於當月第三個週五收盤後生效。被動型 ETF（0050、006208）為降低追蹤誤差，會在生效日 **13:25-13:30** 收盤集合競價時段集中換股，造成：

- **被剔除的個股** → 單向賣壓 → 尾盤超跌 → **隔日開盤均值回歸**
- **被納入的個股** → 單向買盤 → 尾盤超漲 → 隔日回落

## 重要發現（回測 14 次）

| 項目 | 數據 |
|------|------|
| 做多超跌股勝率 | **88.89%** |
| 做空超漲股勝率 | **0.00%** |
| 平均單次淨報酬（做多） | **+1.72%** |
| 雙邊交易成本 | ~0.357% |

> ⚠️ **只做多，不做空。** 做空超漲股全部虧損，直接放棄。

## Web UI

內建 Flask Web 介面，支援 PWA（可加入手機主畫面），響應式設計，手機側欄自動收合為漢堡選單。

### 啟動

```bash
python3 run_webui.py
```

開啟 http://localhost:5000

### 功能頁面

| 頁面 | 功能 |
|------|------|
| Dashboard | 今日日期、成本、門檻、下次生效日 |
| Schedule | 季度審核/生效日期表 |
| Adjust | 編輯調整名單，自動比對 0050 持股差異 |
| Monitor | 尾盤 13:25-13:30 即時監控 |
| Check | 手動訊號評估 |
| Positions | 持倉列表 |
| Exit | 平倉 |
| Summary | 績效統計 |

### 自動比對 0050 成分股

Adjust 頁面提供「自動比對」按鈕：

1. 從 **etfinfo.tw** 抓取當前 0050 持股（含權重）
2. 比對上一季度快照（或從 GitHub git history 取得）
3. 自動填入剔除/納入清單
4. 若無變動顯示「本次成分股無變動」

### PWA 部署

將 `wsgi.py` 部署至 PythonAnywhere 等 WSGI 伺服器。`manifest.json` 與 `sw.js` 已內建，加入主畫面即可使用。

## CLI 使用方式

### 0. 自動抓取 0050 持股差異

```bash
python3 run.py auto-compare
```

### 1. 事前準備：輸入本次調整名單

```bash
python3 run.py adjust

=== 輸入本次調整名單 ===
季度 (如 2026Q3): 2026Q3
被剔除的股票代號 (逗號分隔): 1301,2002
新納入的股票代號 (逗號分隔): 3037,3665,8046
權重調降的代號 (選填): 
```

### 2. 查詢下次調整時程

```bash
python3 run.py schedule
```

### 3. 生效日尾盤檢查訊號

```bash
python3 run.py adjust           # 先輸入名單
python3 run.py monitor all      # 批次掃描所有剔除/調降股
```

### 4. 單檔監控

```bash
python3 run.py monitor 5871
```

### 5. 手動輸入數據

```bash
python3 run.py check 5871 -3.9 1.2
```

### 6. 隔日開盤平倉

```bash
python3 run.py exit
```

### 7. 查詢績效統計

```bash
python3 run.py summary
```

### 8. 查詢目前持倉

```bash
python3 run.py positions
```

## 資料來源

- **0050 持倉（主要）**：`etfinfo.tw` → 解析 `__NUXT_DATA__`，即時含權重
- **0050 持倉（備援）**：GitHub `tbdavid2019/stock-index-api` → `fund_0050.json`
- **上一季比對**：本地快照 `tw50_snapshots.json`，或 GitHub commit history
- **股價**：TWSE `STOCK_DAY_ALL`（全市場收盤價） + `getStockInfo.jsp`（即時報價）
- **股票名稱**：TWSE `BWIBBU_ALL`，快取於 `stock_names.json`

## 策略篩選條件

| 條件 | 說明 |
|------|------|
| Z-score ≥ 2.5σ | 尾盤跌幅需超過個股日內波動率的 2.5 倍標準差 |
| 方向 = Long | 僅接受超跌買進（負偏差） |
| 非處置股/注意股 | 處置股分盤撮合破壞套利流動性 |
| 融券比率 < 25% | 避免空單過度擁擠 |

## 交易成本計算

| 項目 | 費率 |
|------|------|
| 手續費（券商 6 折後） | 0.057% |
| 證券交易稅（賣出） | 0.300% |
| **雙邊合計** | **~0.357%** |

> 若無券商折讓，成本將升至 0.585%，會嚴重侵蝕利潤。請確保你有手續費折讓。

## 風險提示

1. **隔夜系統性風險**：美股若隔夜大跌，台股隔日跳空會淹沒均值回歸收益
2. **擁擠交易反噬**：當所有人都預期尾盤會跌而搶進時，可能發生反向甩尾
3. **流動性風險**：張數過大可能造成滑價，小額交易較安全
4. 本程式僅提供**訊號產生與交易記錄**，實際下單請自行操作
